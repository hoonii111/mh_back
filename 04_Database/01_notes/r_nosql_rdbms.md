# NoSQL vs RDBMS 비교

## 1. 기본 개념

| 구분     | RDBMS                   | NoSQL                            |
| ------ | ----------------------- | -------------------------------- |
| 데이터 모델 | 테이블, 행/열 구조             | 문서(Document), 키-값, 컬럼 패밀리, 그래프 등 |
| 스키마    | 엄격한 스키마 필요              | 유연한 스키마(스키마리스)                   |
| 확장성    | 수직 확장(Vertical Scaling) | 수평 확장(Horizontal Scaling)        |

---

## 2. 주요 특징

### RDBMS

* **ACID** 트랜잭션 보장
* SQL을 통한 복잡한 조인 및 쿼리 지원
* 스키마 강제화로 데이터 무결성 우수
* 수직 확장에는 한계 존재

### NoSQL

* **BASE** 특성(가용성, 분할 허용)
* 다양한 데이터 모델 지원
* 대량 데이터 및 트래픽에 유리한 수평 확장
* 복잡한 조인 지원이 약함

---

## 3. 사용 사례

| 유형            | 대표 DB             | 적합한 용도                    |
| ------------- | ----------------- | ------------------------- |
| RDBMS         | MySQL, PostgreSQL | 금융, ERP, CRM 등 데이터 무결성 중시 |
| Document DB   | MongoDB, CouchDB  | 로그, 콘텐츠 관리, 설정·프로필 저장     |
| Key-Value DB  | Redis, DynamoDB   | 캐시, 세션 저장, 빠른 조회 패턴       |
| Column Family | Cassandra, HBase  | 시계열 데이터, 대규모 분석           |
| Graph DB      | Neo4j, JanusGraph | 소셜 그래프, 추천 시스템            |

---

## 4. 선택 기준

1. **데이터 일관성 vs 확장성**

   * 일관성 우선 → RDBMS
   * 확장성·유연성 우선 → NoSQL
2. **쿼리 복잡도**

   * 복잡한 조인/트랜잭션 → RDBMS
   * 단순 조회·문서 저장 → Document DB
3. **성능 요구 사항**

   * 실시간 분석 vs 대규모 쓰기/읽기 트래픽
4. **운영 및 관리**

   * 성숙된 SQL 생태계 vs 신규 기술 학습 비용

---

## 5. 요약

* **RDBMS**: 안정적 일관성, 복잡 쿼리 강점
* **NoSQL**: 유연 스키마, 수평 확장 강점
* 프로젝트 요구사항에 맞춰 적절한 DB 선택이 중요.
